Hello, This is my first page running in Docker!!

Things I had to do to be here!!


1. Get an account on https://hub.docker.com

2. Create a repository >> otorres31/qa-test

3. Link an Account to get Automated Builds (every time the repo has a change an automated build will be triggered), in my case I have linked docker with github >> otorres-ts/qa-test

4. Download Docker for Windows 8.1 (in my case) >> https://docs.docker.com/toolbox/overview/

5. Docker for Windows 10 and above can be found on next link >> https://download.docker.com/win/beta/InstallDocker.msi

6. Create a Dockerfile in the root of the repository >> this will contain the instructions to be run when building, for example, use ubuntu or ngix for the image. You can see next video for details >> https://www.youtube.com/watch?v=LQjaJINkQXY

This could be very complex in my case I did a basic setting.

FROM nginx
COPY wrapper.sh /
COPY html /usr/share/nginx/html
RUN chmod +x wrapper.sh
CMD ["./wrapper.sh"]

7. Get the latest changes from the docker repository, run next docker command:
$ docker pull otorres31/qa-test

8. Run the docker image:
$ docker run --rm otorres31/qa-test
If all goes well you will see >> Nginx is running...

9. Give a name to the port will be used to run the application:
$ docker run -d -P --name qa-test otorres31/qa-test
You will get an id as response:
4b8fdf4d3c23b96ad68c559b2c7c18a3a6186fe748991de15b75357f206901e3

10. Get the port was assigned to the application:
$ docker port qa-test
You will see something similar to >> 80/tcp -> 0.0.0.0:32770 (in Windows 8 or below IP is not displayed with this command, see next step to get that ip)

11. Get the ip where assigned in Windows 8 or below:
$ docker-machine ip default
You will see something similar to >> 192.168.99.100

12. Now you have the ip and the pport where application is running.

13. Go to the browser and go to that ip+port, as in the example above it would be >> http://192.168.99.100:32769/

14. If all goes well, you will see the nginx page:
Welcome to nginx!

If you see this page, the nginx web server is successfully installed and working. Further configuration is required.

For online documentation and support please refer to nginx.org.
Commercial support is available at nginx.com.
Thank you for using nginx.

15. Now, let's go to another html page we have in the repository:
http://192.168.99.100:32770/Test_formWidgetCSS_production.html

16. Page should be displayed properly.

17. Now, go back to my first page in Docker page:
http://192.168.99.100:32770/My_First_Page_In_Docker.html

18. Deleting the image:
$ docker images
$ docker rmi -f or

19. Deleting the container (where the image lives):
$ docker ps
$ docker rm -f or
